# 사용자 프로필

## 사용자 맥락

- 사용자는 비개발자에 가깝고, 사내 포털을 직접 운영합니다.
- 기술 용어보다 결과, 버튼 이름, 파일 위치, 실행 명령이 중요합니다.
- 혼자 운영하는 작은 팀 환경입니다. 안정성과 유지보수가 빠른 기능 추가보다 우선입니다.
- UI 언어는 한국어, 날짜 기준은 Asia/Seoul입니다.

## 커뮤니케이션 원칙

- 한 번에 한 가지 핵심 질문만 합니다.
- 선택지가 필요하면 A/B/C와 추천안을 함께 제시합니다.
- 작업 중에는 짧게 현재 상태를 알려줍니다.
- 위험 작업은 이유와 영향 범위를 설명하고 확인을 받습니다.
- 완료 보고는 변경 사항과 검증 결과 중심으로 짧게 씁니다.

## 피할 것

- 기술 용어만 나열하기
- 사용자가 판단하기 어려운 긴 질문 여러 개 던지기
- 운영 DB, 배포, 삭제 작업을 임의로 진행하기
- “나중에 직접 복사하세요”처럼 같은 워크스페이스에서 불필요한 수동 절차를 요구하기

## 좋은 설명 방식

나쁜 예:

```text
VAPID public key를 Uint8Array로 변환해 applicationServerKey로 전달합니다.
```

좋은 예:

```text
브라우저가 푸시 알림 서버를 알아볼 수 있도록, 공개 키 문자열을 브라우저가 읽는 숫자 배열 형태로 바꿔 전달합니다.
```

## 사용자가 중요하게 보는 기준

- 기존 업무가 끊기지 않는가
- 모바일에서도 쓸 수 있는가
- 관리자와 일반 직원 권한이 섞이지 않는가
- 데이터가 사라지거나 잘못 계산되지 않는가
- 배포 후 바로 확인할 수 있는가
