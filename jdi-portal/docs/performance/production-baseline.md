이 값은 집계된 기준선이며, 개별 사용자·요청·환경 정보를 포함하지 않습니다.

## 반복 가능한 브라우저 확인 시나리오

1. 새 브라우저 프로필에서 포털 로그인 화면을 열고, 대시보드로 이동할 수 있는 승인된 테스트 계정으로 로그인합니다.
2. 대시보드 첫 화면이 안정화될 때까지 기다린 뒤 개발자 도구의 Network에서 초기 문서와 JavaScript 전송량을 확인합니다.
3. 인플루언서 화면을 처음 열고, 콜드 로드 전송량과 JavaScript 전송량을 기록합니다. 초기 그래프에 XLSX 청크가 포함되지 않는지 확인합니다.
4. 업무와 일정 화면을 각각 처음 열고, 화면을 마운트한 직후 반복 새로고침 요청이나 전역 사전 로드 요청이 없는지 확인합니다.
5. 같은 시나리오를 캐시를 비운 새 브라우저 프로필에서 다시 실행하고, 결과는 집계 수치로만 기록합니다.

## 자동 점검

빌드 후 아래 명령으로 각 라우트의 초기 JavaScript 크기, 라우트 예산, 인플루언서 초기 XLSX 그래프, 정적 회귀 보호 규칙을 검사합니다.

```bash
npm run perf:audit
npm run test:performance
```
