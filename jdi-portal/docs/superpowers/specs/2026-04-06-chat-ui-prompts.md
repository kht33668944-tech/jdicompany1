# 채팅 UI 프롬프트 (Variant AI용)

> 아래 프롬프트를 Variant AI에 하나씩 넣어서 UI를 생성하세요.
> 각 프롬프트는 독립적으로 사용할 수 있습니다.

---

## 디자인 시스템 참고 (공통으로 앞에 붙여주세요)

```
디자인 시스템:
- 브랜드 컬러: Blue (#2563eb 기반)
- 배경: 깨끗한 화이트 베이스, 연한 slate/gray 톤의 서브 배경
- 카드: 흰색, rounded-2xl~3xl, 가벼운 shadow
- 폰트: 깔끔한 산세리프, 한국어 UI
- 아이콘: Phosphor Icons 스타일 (가벼운 선형 아이콘)
- 전체 톤: 미니멀하고 깔끔한 SaaS 느낌, 글래스모피즘 최소한 활용
- 모바일 퍼스트 반응형
```

## 프롬프트 1: 채팅 메인 페이지 (데스크탑)

```
사내 포털 채팅 페이지를 디자인해줘. 4~10명이 쓰는 소규모 회사 내부 메신저야.

레이아웃:
- 왼쪽에 채널 목록 사이드 패널, 오른쪽에 대화창
- 사이드 패널과 대화창이 자연스럽게 나뉘어진 2단 구성

왼쪽 채널 목록 패널:
- 상단에 "채팅" 타이틀 + 채널 추가 버튼 (+ 아이콘)
- 검색 입력창 (채널/메시지 검색용)
- "나만의 메모" 채널이 맨 위에 고정 (메모장 아이콘, 다른 채널과 시각적으로 구분)
- 아래로 채널 목록 나열
  - 각 채널: 채널 이름, 마지막 메시지 미리보기 (1줄), 시간, 읽지 않은 메시지 수 뱃지
  - 현재 선택된 채널은 배경색으로 하이라이트
  - 채널 목록은 최신 메시지 순으로 정렬

오른쪽 대화창:
- 상단 헤더: 채널명, 멤버 수 표시, 오른쪽에 검색/서랍/설정 아이콘
- 중앙 메시지 영역:
  - 날짜 구분선 ("2026년 4월 6일 월요일")
  - 메시지: 프로필 아바타 + 이름 + 시간 + 내용
  - 내 메시지는 오른쪽 정렬, 상대 메시지는 왼쪽 정렬
  - 읽음 표시: 메시지 아래에 작게 "3명 읽음" (클릭 가능한 느낌)
  - 시스템 메시지는 중앙 정렬 작은 회색 텍스트 ("○○님이 입장했습니다")
- 하단 입력창:
  - 왼쪽에 파일 첨부 버튼 (클립 아이콘)
  - 넓은 텍스트 입력 영역
  - 오른쪽에 전송 버튼
  - 드래그앤드롭 영역 힌트가 자연스럽게 어울리게

채널을 아직 선택하지 않았을 때:
- 오른쪽 대화창 영역에 빈 상태 일러스트 + "채널을 선택하거나 새 채널을 만들어보세요" 안내

분위기: Slack이나 Discord 같은 무거운 느낌이 아니라, 깔끔하고 가벼운 사내 메신저 느낌. 공간감 있고 여백이 적절한 디자인.
```

## 프롬프트 2: 채팅 메인 페이지 (모바일)

```
위에서 만든 사내 채팅 페이지의 모바일 버전을 디자인해줘.

모바일에서는 2단 레이아웃이 아니라 화면 전환 방식:

화면 1 — 채널 목록 (기본 화면):
- 상단: "채팅" 타이틀 + 채널 추가 버튼
- 검색 입력창
- 나만의 메모 (고정, 상단)
- 채널 목록 (풀 너비)
  - 각 항목: 채널 아이콘/이니셜 + 채널명 + 마지막 메시지 + 시간 + 뱃지
  - 스와이프 가능한 느낌

화면 2 — 대화방 (채널 선택 시):
- 상단: 뒤로가기 화살표 + 채널명 + 멤버 수 + 햄버거/설정 아이콘
- 메시지 영역 (풀스크린)
- 하단 입력창: 첨부 + 텍스트 + 전송
  - 키보드가 올라와도 자연스럽게 대응하는 느낌

모바일 터치 타겟은 최소 44px. 전체적으로 카카오톡처럼 익숙하면서도 좀 더 깔끔한 SaaS 느낌으로.
```

## 프롬프트 3: 채널 생성 모달

```
채팅 채널을 새로 만드는 모달을 디자인해줘.

모달 내용:
- 타이틀: "새 채널 만들기"
- 채널 이름 입력 (필수)
- 채널 설명 입력 (선택, 짧은 한 줄)
- 멤버 선택 영역:
  - 직원 목록에서 체크박스로 멤버 선택
  - 각 직원: 프로필 아바타 + 이름 + 역할(직급)
  - 선택된 멤버는 상단에 작은 아바타 칩으로 표시
  - 직원 검색 가능
- 하단: 취소 / 만들기 버튼

4~10명 규모라 직원 목록이 짧으니까 심플하게.
모달은 중앙 정렬, 적당한 너비 (max-width 480px 정도), 부드러운 라운딩.
```

## 프롬프트 4: 채널 설정 / 멤버 관리 드로어

```
채팅 채널의 설정 및 멤버 관리를 위한 오른쪽 슬라이드 드로어를 디자인해줘.

드로어 내용:
- 상단: 채널명 (수정 가능) + 닫기 버튼
- 채널 설명 (수정 가능)
- 구분선
- 멤버 목록 섹션:
  - "멤버 (4명)" 헤더 + "초대하기" 버튼
  - 멤버 리스트: 아바타 + 이름 + 역할 뱃지(owner/member)
  - 각 멤버 옆에 "내보내기" 옵션 (더보기 메뉴 또는 작은 버튼)
  - 나 자신에게는 "나가기" 표시
- 구분선
- 하단 위험 영역:
  - "채널 나가기" 버튼 (빨간 텍스트)
  - "채널 삭제" 버튼 (생성자에게만 표시, 빨간 배경)

오른쪽에서 슬라이드되어 나오는 패널. 
배경은 흰색, 부드러운 그림자. 너비는 360px 정도.
```

## 프롬프트 5: 채팅방 서랍 (사진/파일/링크)

```
카카오톡의 "채팅방 서랍" 같은 기능을 웹으로 디자인해줘.
채팅방에서 공유된 미디어/파일/링크를 분류해서 모아보는 패널이야.

레이아웃:
- 오른쪽에서 슬라이드되어 나오는 넓은 패널 (또는 전체 화면 오버레이)
- 상단: "채팅방 서랍" 타이틀 + 닫기 버튼
- 탭 3개: 사진 | 파일 | 링크

사진 탭:
- 월별로 그룹핑 ("2026년 4월", "2026년 3월")
- 사진을 그리드 형태로 나열 (3~4열)
- 각 사진은 정사각형 썸네일
- 클릭하면 원본 보기
- 상단에 "전체 다운로드" 버튼

파일 탭:
- 파일 리스트 형태
- 각 항목: 파일 아이콘 + 파일명 + 크기 + 보낸 사람 + 날짜
- 다운로드 버튼
- 월별 그룹핑

링크 탭:
- 공유된 URL 리스트
- 각 항목: URL 미리보기 (OG 이미지 + 제목 + 설명) 또는 단순 링크
- 클릭하면 새 탭에서 열기

카카오톡 서랍을 참고하되, 웹 SaaS에 맞게 더 깔끔하고 넓은 공간감으로 디자인해줘.
```

## 프롬프트 6: 이미지 묶어보내기 + 미리보기

```
채팅에서 이미지를 여러 장 보냈을 때의 메시지 표시 방식을 디자인해줘.

이미지 묶어보내기:
- 1장: 그냥 인라인 이미지 (적당한 크기, 라운딩)
- 2장: 가로 2열 그리드
- 3장: 큰 이미지 1개 + 작은 이미지 2개 조합
- 4장 이상: 2x2 그리드, 4장 초과 시 마지막에 "+3" 오버레이

이미지 클릭 시:
- 전체 화면 오버레이로 원본 이미지 표시
- 좌우 화살표로 같은 묶음의 다른 이미지 탐색
- 하단에 작은 썸네일 네비게이션
- 다운로드 버튼, 닫기 버튼
- 배경은 어두운 딤 처리

카카오톡이나 텔레그램의 이미지 그룹 메시지를 참고하되, 
웹에 맞게 좀 더 여유있는 레이아웃으로 디자인해줘.
```

## 프롬프트 7: 나만의 메모 채널

```
채팅 앱 안에 "나만의 메모" 기능을 디자인해줘.
카카오톡의 "나에게 보내기" 같은 개인 메모 공간이야.

특징:
- 채널 목록에서 맨 위에 고정, 다른 채널과 시각적으로 구분
  (예: 다른 배경색, 메모장 아이콘, "나만의 메모" 라벨)
- 대화방에 들어가면 일반 채팅과 같은 UI지만:
  - 상대방이 없으니 내 메시지만 왼쪽 정렬로 쭉 나열
  - 읽음 표시 없음
  - 헤더에 "나만의 메모" + 메모장 아이콘
- 텍스트, 이미지, 파일 모두 보낼 수 있음
- 삭제 가능, 검색 가능

분위기: 개인 메모장/스크랩북 느낌. 약간 따뜻한 톤이나 차별화된 배경색을 써서
일반 채팅방과 구분되면 좋겠어.
```

## 프롬프트 8: 빈 상태 / 온보딩

```
채팅 기능의 빈 상태(empty state) 화면들을 디자인해줘.

상태 1 — 채널 미선택 (데스크탑 오른쪽 영역):
- 심플한 일러스트 또는 아이콘
- "채널을 선택하거나 새 채널을 만들어보세요"
- 채널 만들기 버튼 (선택사항)

상태 2 — 채널 목록이 비어있음 (첫 사용):
- 환영 메시지: "채팅을 시작해보세요!"
- 간단한 안내: "채널을 만들어 팀원들과 대화하세요"
- "첫 채널 만들기" 버튼

상태 3 — 대화방에 메시지가 없음 (새 채널):
- "첫 메시지를 보내보세요!" 안내
- 가볍고 친근한 일러스트

각 빈 상태는 밝고 친근한 톤으로, 사용자가 다음 행동을 자연스럽게 할 수 있게 유도해줘.
```

## 사용 방법

1. **공통 디자인 시스템**을 먼저 Variant AI에 컨텍스트로 제공
2. **프롬프트 1**부터 순서대로 생성
3. 마음에 드는 결과가 나오면 다음 프롬프트로 이동
4. 필요에 따라 프롬프트를 수정하거나 조합해서 사용

### 추천 순서

```
프롬프트 1 (데스크탑 메인) → 2 (모바일) → 3 (채널 생성)
→ 7 (메모) → 8 (빈 상태) → 4 (설정 드로어)
→ 6 (이미지 묶음) → 5 (서랍)
```
